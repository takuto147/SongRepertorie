# 🎤 カラオケメモアプリ 要件定義書（改訂版）

## 1. プロジェクト概要
カラオケ好きが自分のレパートリーや練習状況を記録・管理できるアプリ。  
歌いたい曲を忘れないように記録でき、キーや得意度、点数なども管理可能。  
困ったときにはランダムに選曲してくれるお楽しみ機能も搭載。

## 2. ユーザー定義
- 趣味でカラオケに通う人
- 自分の歌の傾向や上達を記録・管理したい人
- 点数やキー情報をメモしてレパートリーを増やしたい人

## 3. 機能要件

### 3.1 ユーザー管理（任意）
- ユーザー登録／ログイン（MySQLで認証情報を管理）
  - パスワードは bcrypt 等でハッシュ化
  - JWT でログイン状態を管理し、セッションレスなAPI設計
- 自分の登録曲・データのみを閲覧・編集可能

### 3.2 曲管理
- 曲の登録（曲名、アーティスト、メモ）
- 練習ステータスの管理（例：練習中／得意曲など）
- 歌唱キー、カラオケ点数、自由メモの記録
- 歌詞冒頭の取得（Musixmatch APIを使用）
- 登録時の曲検索、歌手検索（Spotify APIを使用予定）

### 3.3 検索・表示
- 登録曲の一覧表示
- キーワード検索（曲名・アーティスト）
- ソート機能（曲名順／アーティスト順／点数順 など）
- フィルタ機能（得意曲だけ、練習中だけ など）

### 3.4 ユーティリティ機能
- ランダム選曲機能：登録曲の中からランダムに1曲を提案
- 外部API連携による補助入力
  - Musixmatch API：歌詞の冒頭表示
  - Spotify API：曲・アーティスト情報の補完や検索支援

## 4. 非機能要件
- モバイルファースト対応（スマートフォンでの利用を優先設計）
- 5秒以内にデータを読み込み完了（キャッシュ活用、非同期取得）
- ログインユーザーのデータのみ表示（セキュリティを意識した認可制御）
- 日本語環境への最適化（歌詞・文字コード対応）

## 5. 技術要件
- フロントエンド：React
- バックエンド：Java（Spring Boot）
- データベース：MySQL
- 認証方式：JWTベースのトークン認証
- 外部API連携：
  - Musixmatch API（歌詞取得／歌い出し表示）
  - Spotify API（曲・アーティストの正式情報取得、検索補助）
