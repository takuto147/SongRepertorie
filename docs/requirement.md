# 🎤 カラオケメモアプリ 要件定義書

## 1. プロジェクト概要
カラオケ好きが自分のレパートリーや練習状況を記録・管理できるアプリ。

## 2. ユーザー定義
- 趣味でカラオケに通う20〜40代ユーザー

## 3. 機能要件
### 3.1 ユーザー管理（任意）
- ユーザー登録／ログイン（なければ省略可）

### 3.2 曲管理
- 曲の登録（曲名、アーティスト、メモ）
- 練習ステータスの管理（練習中、得意曲など）

### 3.3 検索・表示
- 登録曲の一覧表示
- 曲の検索・フィルター・ソート機能

## 4. 非機能要件
- モバイルファースト対応
- 5秒以内にデータ読み込み
- ログインユーザーのデータのみ表示（ユーザー管理する場合）

## 5. 技術要件
- フロントエンド：React (Cursorで開発)
- バックエンド：Spring Boot
- データベース：MySQL or PostgreSQL
